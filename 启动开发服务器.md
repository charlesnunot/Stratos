# 启动开发服务器

## 当前状态

开发服务器正在启动中...

## 访问地址

- **本地地址**：http://localhost:3000
- 如果3000端口被占用，Next.js会自动使用3001、3002等端口

## 如果启动失败

### 问题1：权限错误（EPERM）

**解决方法**：
1. 以管理员身份运行PowerShell
2. 或者关闭所有Node进程后重试

### 问题2：端口被占用

**解决方法**：
```powershell
# 查看占用3000端口的进程
netstat -ano | findstr :3000

# 停止占用端口的进程（替换PID为实际进程ID）
Stop-Process -Id <PID> -Force
```

### 问题3：依赖问题

**解决方法**：
```powershell
# 重新安装依赖
npm install

# 清除缓存
npm cache clean --force
```

## 手动启动

如果自动启动失败，可以手动运行：

```powershell
cd C:\Users\admin\Desktop\Stratos
npm run dev
```

## 检查服务器状态

服务器启动成功后，您会看到类似这样的输出：

```
▲ Next.js 14.0.4
- Local:        http://localhost:3000
- Ready in 2.5s
```

然后就可以在浏览器中访问 http://localhost:3000 了！
