# 支持聊天功能决策文档

## 决策：暂不实现支持聊天功能

### 背景

项目中存在 `conversation_type = 'support'` 的数据库支持，但前端无 UI 实现。

### 决策理由

1. **工单系统已满足需求**
   - 工单系统提供了完整的客服支持流程
   - 支持异步沟通，适合复杂问题
   - 已有完整的通知、分配、状态管理机制

2. **实现成本 vs 收益**
   - 实现支持聊天需要：
     - 创建支持会话的 UI
     - 实时消息推送
     - 支持人员分配机制
     - 与现有聊天系统集成
   - 工单系统已能覆盖大部分支持场景

3. **用户体验考虑**
   - 工单系统更适合：
     - 需要记录和追踪的问题
     - 需要多轮沟通的复杂问题
     - 需要分配给不同支持人员的问题
   - 即时聊天更适合简单快速的问题，但可能缺乏追踪性

### 建议

**当前阶段**：专注于完善工单系统功能，包括：
- ✅ 工单通知（已完成）
- ✅ 工单分配（已完成）
- ✅ 工单类型和状态管理（已完成）
- ✅ 关键页面的客服入口（已完成）

**未来考虑**：如果用户反馈需要即时聊天支持，可以考虑：
1. 在工单详情页集成简单的即时消息功能
2. 或创建独立的支持聊天界面
3. 但需要明确与工单系统的关系（是否关联、如何转换等）

### 数据库清理建议

如果确定不使用支持聊天功能，可以考虑：
1. 保留 `conversation_type = 'support'` 字段（不影响现有功能）
2. 或在未来迁移中移除该选项（如果确定不需要）

### 结论

**暂不实现支持聊天功能，继续使用和完善工单系统。**
