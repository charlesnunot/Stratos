# 编辑页多语言修复 - 01 问题分析

**问题**：访问 `/en/seller/products/[id]/edit` 时，页面出现中文，且商品数据全部显示为中文。

**目标**：在 `/en/` 路由下，界面和商品数据应优先显示英文；在 `/zh/` 下显示中文。

---

## 问题来源

| 类型 | 描述 | 示例 |
|------|------|------|
| UI 硬编码中文 | 未使用 `t()`，直接写死中文 | `商品图片`、`现有图片`、`问题`、`答案`、`已售罄` |
| 中文 fallback | `t('key') \|\| '中文'` 在 key 缺失时显示中文 | `t('productDetailsPlaceholder') \|\| '例如：...'` |
| 商品数据未按 locale 选择 | 始终用 `product.name`、`product.description`，未使用 `name_translated` 等 | 原文为中文时，英文页也显示中文 |
| Toast 标题硬编码 | `title: '错误'` | 错误提示弹窗始终为中文 |

---

## 商品翻译字段（数据库）

- `content_lang`：原文语言（'zh' | 'en'）
- `name` / `name_translated`：商品名
- `description` / `description_translated`：商品描述
- `details` / `details_translated`：商品详情
- `faq` / `faq_translated`：FAQ 数组

已有 `getDisplayContent(locale, contentLang, content, contentTranslated)` 用于单段文本，编辑页需复用此逻辑。
