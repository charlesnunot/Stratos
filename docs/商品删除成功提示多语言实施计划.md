# 商品删除成功提示多语言实施计划

**目标**：将商品删除成功 toast 的硬编码中文替换为翻译键，支持中英文环境。

---

## 一、现状

**文件**：`src/app/[locale]/(main)/seller/products/page.tsx`

**当前代码**（约 73-77 行）：

```tsx
toast({
  variant: 'success',
  title: '已删除',
  description: '商品已删除',
})
```

**问题**：`title` 与 `description` 为硬编码中文，英文环境下仍显示中文。

---

## 二、涉及文件

| 文件 | 修改内容 |
|------|----------|
| `src/messages/zh.json` | seller 命名空间新增 2 个翻译键 |
| `src/messages/en.json` | seller 命名空间新增 2 个翻译键 |
| `src/app/[locale]/(main)/seller/products/page.tsx` | 将硬编码文案替换为 `t(...)` |

---

## 三、实施步骤

### 步骤 1：补充 zh.json 翻译

**文件**：`src/messages/zh.json`  
**位置**：`seller` 命名空间内（建议放在 `updateFailed` 附近或 seller 块末尾）

```json
"deleteSuccess": "已删除",
"productDeleted": "商品已删除"
```

### 步骤 2：补充 en.json 翻译

**文件**：`src/messages/en.json`  
**位置**：`seller` 命名空间内，与 zh.json 对应位置

```json
"deleteSuccess": "Deleted",
"productDeleted": "Product deleted"
```

### 步骤 3：修改 page.tsx

**文件**：`src/app/[locale]/(main)/seller/products/page.tsx`

**修改前**：

```tsx
toast({
  variant: 'success',
  title: '已删除',
  description: '商品已删除',
})
```

**修改后**：

```tsx
toast({
  variant: 'success',
  title: t('deleteSuccess'),
  description: t('productDeleted'),
})
```

**说明**：页面已使用 `const t = useTranslations('seller')`，无需新增导入。

---

## 四、实施顺序

| 顺序 | 操作 |
|------|------|
| 1 | zh.json 新增 deleteSuccess、productDeleted |
| 2 | en.json 新增 deleteSuccess、productDeleted |
| 3 | page.tsx 替换 toast 文案为 t('deleteSuccess')、t('productDeleted') |

---

## 五、验证步骤

1. 切换 locale 为 `zh`，删除商品，确认 toast 显示「已删除」「商品已删除」
2. 切换 locale 为 `en`，删除商品，确认 toast 显示「Deleted」「Product deleted」
3. 确认删除流程与原有逻辑一致
